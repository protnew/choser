## 🔴 Критические проблемы

### К1. Grid.jsx — 1 007 строк (God Object)

**Файл:** `src/components/Grid.jsx` (53 KB)

Крупнейший компонент, который делает **всё**: рендеринг таблицы, AG Grid конфигурация, EBM-вкладки, карточки, экспорт, tooltips, темизация, responsive, SEO. Любое изменение — риск сломать 20 других вещей.

**Рекомендация:** Разбить на 8-10 компонентов:
- `GridContainer.jsx` — оркестратор (~100 строк)
- `GridTable.jsx` — AG Grid рендеринг (~200 строк)
- `GridColumnConfig.js` — конфигурация колонок (~150 строк)
- `GridToolbar.jsx` — панель инструментов (~100 строк)
- `GridEBMPanel.jsx` — уже частично вынесен
- `GridExport.jsx` — логика экспорта (~100 строк)
- `GridMobile.jsx` — мобильная адаптация (~100 строк)
- `GridState.js` — custom hook для стейта (~100 строк)

**Оценка:** 16h (рефакторинг + QA)

### К2. Ноль тестов

Нет ни одного автотеста в пайплайне. Файлы `test-*.js` — ручные скрипты. При 12K строк кода — критический риск.

**Рекомендация:**
- Vitest для unit-тестов (уже есть Vite)
- Приоритет: `calc.js`, `ebm.js`, `ebm-bayesian.js`, `statistics.js` (чистые функции → 100% покрытие за 1 день)
- Потом: API endpoints через `app.request()` от Hono
- Потом: React Testing Library для критических UI-компонентов

**Оценка:** 24h на MVP coverage (утилиты + API)

### К3. `.gitignore` игнорирует ВСЕ `.md` файлы

```
*.md
!README.md
!AG_RULES.md
```

FEATURES.md, PROGRESS.md, DEEP_ANALYSIS, HANDOVER — **НЕ в git**. Теряем документацию при потере диска.

**Рекомендация:** убрать `*.md` из `.gitignore`, добавить явные исключения:
```
# *.md — НЕ игнорировать (документация в репозитории)
node_modules
.wrangler
dist
*.log
analysis/
*.sqlite*
*.db*
```

**Оценка:** 0.5h

### К4. 30 MB SQL-файлов в репозитории

`seed_data.sql` (6.3 MB), `update_rows_grades.sql` (5.5 MB) и т.д. Хранить генерируемые данные в git — раздувать клон.

**Рекомендация:**
- Добавить `*.sql` в `.gitignore` (кроме `schema.sql`, `seed_choser.sql`)
- Хранить бэкапы в `backup/` (уже gitignored) или в облаке (S3/R2)

**Оценка:** 1h

### К5. Смешанные среды: Cloudflare Workers + Docker

`package.json` содержит скрипты Wrangler (Cloudflare), но Dockerfile собирает standalone Node.js сервер. Две параллельные архитектуры.

**Рекомендация:** Выбрать одну:
- **Docker** для self-hosted (уже работает)
- Удалить Wrangler-зависимости или вынести в отдельную ветку

**Оценка:** 4h на очистку

## 🟡 Средние проблемы

### С1. ai_service.js — 589 строк (слишком много для одного файла)

Генерация таблиц, критика, similar-генерация, fallback chain — всё в одном объекте `AI_SERVICE`.

**Рекомендация:** Разбить:
- `llm-router.js` — fallback chain, circuit breaker, cache
- `table-generator.js` — генерация таблиц
- `table-critic.js` — оценка качества
- `research-agent.js` — deep research

### С2. auth.js — минимальная JWT-реализация

Нет refresh token, нет rate limiting на login, нет password validation. `AUTH_ENABLED=false` по умолчанию.

**Рекомендация:**
- Refresh token flow (7d TTL, httpOnly cookie)
- Rate limit на `/auth/login` (5 attempts / 15 min)
- Password strength validation
- `@hono/zod-validator` для входных данных (уже в зависимостях)

### С3. CORS `*` по умолчанию

`CORS_ORIGINS=${CORS_ORIGINS:-*}` — любой сайт может вызвать API. Для AI endpoints — проблема (cost per call).

**Рекомендация:** Разделить:
- `/api/tables/*` — public, CORS *
- `/api/admin/*`, `/mcp/*` — restricted, CORS по whitelist
- `/api/ai/*` — API key required

### С4. MCP Server — 391 строка, нет typed schema

MCP tools описаны inline без валидации входных параметров.

**Рекомендация:** Zod-схемы для каждого MCP tool input (Zod уже в зависимостях).

### С5. Нет error monitoring

`console.error` — единственный способ логирования.

**Рекомендация:** Pino (уже установлен в EDP) для structured JSON логов.

## 💾 Резюме

Choser — проект с **уникальным набором фич** (EBM + Council + MCP), который страдает от типичных проблем быстрого прототипирования: God Object, нет тестов, смешанные среды, раздутый репозиторий. Бизнес-логика сильная, инфраструктура — нет.

**Приоритет №1:** тесты на математику (`ebm.js`, `ebm-bayesian.js`) — если там баг, вся EBM-аналитика неверна.
**Приоритет №2:** `.gitignore` — иначе потеряете документацию.
**Приоритет №3:** рефакторинг Grid.jsx — иначе каждая новая фича будет ломать три других.

# Council Search Modes — 3 тарифа

**Дата:** 2026-05-24

## Тариф 1: 💰 Из памяти модели (без поиска)
- **Стоимость:** только токены LLM
- **Механика:** агенты отвечают из training data
- **Скорость:** быстрая (60-80 сек × кол-во агентов)
- **Точность:** низкая — могут быть устаревшие данные
- **Для кого:** экспресс-оценка, когда точность не критична

## Тариф 2: 🚀 Backend search (один поиск для всех)
- **Стоимость:** 1× search API + токены LLM
- **Механика:**
  1. Один запрос к search API по теме
  2. Результат (топ-5 страниц) вставляется в промпт каждого агента
  3. Агенты оценивают на основе свежих данных
- **Скорость:** +5-10 сек на поиск
- **Точность:** средняя — один источник для всех
- **Для кого:** базовый вариант с актуальными данными

## Тариф 3: 👑 Multi-agent search (каждый ищет под своим углом)
- **Стоимость:** 10× search API + токены LLM (в 2-3 раза дороже)
- **Механика:**
  1. Каждый агент формирует свой поисковый запрос по роли:
     - CEO: "LLM market leaders ROI 2026"
     - CFO: "LLM pricing TCO hidden costs"
     - CISO: "LLM data privacy compliance"
     - CTO: "LLM API context window benchmarks"
     - etc.
  2. Каждый агент ищет → заполняет свою таблицу оценок
  3. Редактор пробегает все таблицы → объединяет в финальную
  4. Результат: N таблиц от агентов + 1 общая от редактора
- **Скорость:** медленная (поиск + LLM × 10 агентов)
- **Точность:** максимальная — разные углы + верификация редактором
- **Для кого:** важные решения с бюджетом

## Параметр API

```json
{
  "searchMode": "none" | "single" | "multi",
  "topic": "...",
  "personas": ["ceo", "cfo"]
}
```

## UI

Три кнопки вместо текущего выбора режима:
```
💰 Без поиска    🚀 Быстрый поиск    👑 Глубокий анализ
  (из памяти)     (1 запрос)          (каждый агент ищет)
```
